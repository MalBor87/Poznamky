Virtualizácia

Inštalácia KVM

Skontrolujte podporu virtualizácie na Ubuntu

egrep -c '(vmx|svm)' /proc/cpuinfo

Ak príkaz vráti hodnotu 0, váš procesor nedokáže spustiť KVM. Teraz skontrolujte, či váš systém môže používať akceleráciu KVM.

sudo kvm-ok

Ak kvm-ok vráti chybu, že nie je možné použiť akceleráciu KVM, je potrebne vykonať inštaláciu cpu-checker.

sudo apt install cpu-checker

reštartujte server Následne aktualizujte repozitáre

sudo apt update

Potom nainštalujte základné balíky KVM pomocou nasledujúceho príkazu

sudo apt install qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils

Virtuálne počítače môžu spúšťať iba členovia používateľských skupín libvirt a kvm. Pridajte používateľa do skupiny libvirt zadaním:

sudo adduser meno libvirt

Teraz urobte to isté pre skupinu kvm

sudo adduser meno kvm

Overte inštaláciu

sudo virsh list

Alebo použite príkaz systemctl na kontrolu stavu libvirtd

sudo systemctl status libvirtd

(na zobrazovanie virtualizacie doplnok v cockpilote) - nutne sa odhlasiť a znovu prihlásiť

sudo apt-get install cockpit-machines

virt-manager

Ak virtualizačný nastroj nie je aktívny, aktivujte ho nasledujúcim príkazom

sudo systemctl enable --now libvirtd

Vytvorenie virtuálneho počítača na Ubuntu 20.04 Na správu virtuálnych PC je možne použiť graficky nastroj virt-manager, alebo požiť príkazový riadok. Nainštalujte virt-manager, nástroj na vytváranie a správu virtuálnych počítačov, nastroj funguje na systémoch z grafickým rozhraním

sudo apt install virt-manager
